**Техническое задание на проект: *Информационно-аналитическая система для любителей напитков с целью анализа состава и популярности коктейлей***

# Цель проекта
Создание системы для просмотра и подбора рецептов коктейлей. Система помогает пользователям подобрать рецепт коктейлей по имеющимся ингредиентам, а также рекомендует напитки на основе их популярности

# Роли пользователей
- **Гость:** просмотр всех доступных рецептов, доступ к информации о коктейлях и ингредиентах
- **Зарегистрированный пользователь:** Добавление рецептов в "Избранное"

# Технический стек
- **Frontend+Backend:** Django
- **База данных:** MySQL
- **Внешние API:** TheCocktailDB API (загрузка рецептов и ингредиентов)
- **Аналитика данных:** Pandas

# Ключевой функционал:
- **Каталог и поиск:** пользователь может искать и просматривать рецепты напитков, ингредиентов
- **Избранное:** авторизованные пользователи могут сохранять понравившиеся рецепты в избранное
- **Рейтинг популярности:** на основе просмотров пользователей формируется рейтинг популярных напитков

# Use Cases
## 1. Поиск напитка по названию
1. Пользователь вводит название или часть названия напитка
2. Система выдаёт результаты вхождения слова/словосочетания в название напитка
3. Пользователь выбирает нужный рецепт из списка результатов

## 2. Поиск напитка по ингредиентам
1. Пользователь выбирает в форме имеющиеся ингредиенты и нажимает кнопку поиска
2. Система выдаёт рецепты, в которые входят указанные ингредиенты
3. Пользователь выбирает нужный рецепт из списка результатов

## 3. Просмотр рецепта коктейля
1. Пользователь открывает страницу рецепта
2. Система фиксирует просмотр для дальнейшей аналитики
3. Система отображает состав, инструкцию, изображение

## 4. Формирование рейтинга популярных коктейлей
1. Система выгружает данные просмотров коктейлей
2. Pandas считает количество просмотров
3. Обновляется информация о популярных напитках
4. Пользователям отображается список популярных напитков

## 5. Добавление рецепта в избранное
0. *ПРЕДУСЛОВИЕ: пользователь авторизовался на сайте*
1. Пользователь выбрал нужный рецепт
2. Пользователь нажал на кнопку добавления в избранное
3. Система сохранила изменения в избранных напитках

## 6. Просмотр избранных напитков
0. *ПРЕДУСЛОВИЕ: пользователь авторизовался на сайте*
1. Пользователь нажал на кнопку перехода в список избранного
2. Система отобразила все избранные напитки пользователя

# Получение данных с API
- Загрузка данных в БД происходит через скрипт парсинга открытого TheCocktailDB API
- Система получает данные о коктейлях и ингредиентах в формате JSON
- Система парсит JSON и сохраняет информацию в БД
